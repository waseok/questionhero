# 안전 탐사 지도 웹앱 디자인 프롬프트

이 문서는 안전 탐사 지도 웹앱의 UI/UX를 일관된 스타일로 생성하거나 리디자인할 때 사용하는 프롬프트 템플릿입니다.
Tailwind CSS 기반 프론트엔드를 전제로 작성되었습니다.

---

## 1) 기본 디자인 프롬프트 (마스터)

아래 문장을 디자인 생성 도구(또는 AI)에게 그대로 전달해 시작하세요.

```md
당신은 공공안전/재난 대응 도메인에 강한 시니어 프로덕트 디자이너입니다.
다음 요구사항에 맞춰 "안전 탐사 지도 웹앱"의 웹 UI를 디자인하세요.

[프로젝트 성격]
- 서비스: 안전 탐사 지도 웹앱
- 사용자: 시민, 탐사팀 운영자, 관리자
- 핵심 목적: 위험 지역 파악, 안전 경로 확인, 제보/점검 상태 추적

[디자인 톤앤매너]
- 신뢰감, 명확성, 즉시성, 과도한 장식 배제
- 공공 서비스 느낌 + 현대적 SaaS UI 균형
- 모바일 우선, 데스크톱 확장

[비주얼 스타일]
- 지도 중심 레이아웃 (Map-first)
- 패널/카드 기반 정보 구조
- 상태 기반 색상 시스템:
  - 안전(Safe): 녹색 계열
  - 주의(Caution): 주황/노랑 계열
  - 위험(Danger): 빨강 계열
  - 정보(Info): 파랑 계열
- 배경은 저채도 중립색, 강조는 상태색으로 제한
- 아이콘은 라인 아이콘 위주, 의미가 즉시 전달되어야 함

[타이포그래피]
- 가독성 최우선 (한글 UI 기준)
- 제목/본문/캡션의 계층을 명확히 분리
- 숫자(거리, 시간, 위험도 점수)는 시각적으로 빠르게 구분되게 강조

[핵심 화면]
1. 메인 지도 화면
2. 위험 지역 상세 패널
3. 안전 경로 탐색 결과 화면
4. 제보 등록/수정 화면
5. 대시보드(요약 지표) 화면

[핵심 컴포넌트]
- 상단 상태바(현재 지역, 알림, 필터 진입)
- 지도 오버레이 컨트롤(줌, 현재위치, 레이어 토글)
- 필터 바(위험도, 시간대, 시설 유형)
- 상태 배지(안전/주의/위험)
- 우측 또는 하단 슬라이딩 상세 패널
- CTA 버튼(경로 안내 시작, 제보하기)
- 토스트/배너 알림(긴급 경보 포함)

[인터랙션 원칙]
- 사용자 행동 1~2번 클릭 내 주요 작업 완료
- 상태 변화는 즉시 피드백 (로딩, 성공, 실패)
- 애니메이션은 짧고 절제되게 (150~250ms)
- 위험 관련 액션은 항상 재확인 단계 제공

[접근성]
- 색상 대비 WCAG AA 이상
- 색상만으로 상태를 구분하지 않기 (아이콘/텍스트 병행)
- 키보드 포커스 가시성 확보
- 스크린리더용 라벨 고려

[반응형]
- 모바일: 지도 + 바텀시트 중심
- 태블릿: 지도 + 반폭 패널
- 데스크톱: 지도 + 고정 사이드 패널 + 요약 위젯

[산출물 형식]
- 화면 구조 설명
- 컬러/타입/간격 토큰 제안
- 주요 컴포넌트 목록과 상태(기본/호버/활성/비활성/에러)
- 사용자 플로우(위험 탐지 → 상세 확인 → 경로 탐색 → 제보)
- 구현 친화적 설명(Tailwind 유틸리티 관점)
```

## 2) 화면별 프롬프트 템플릿

아래 템플릿을 상황에 맞게 복사해 사용하세요.

### 2-1. 메인 지도 화면

```md
안전 탐사 지도 웹앱의 "메인 지도 화면"을 설계하세요.
요구사항:
- 첫 진입 시 사용자가 현재 위험 상태를 3초 이내 이해
- 지도 위 핵심 정보 밀도는 높되 시각적 혼잡 최소화
- 위험도 범례, 필터, 현재 위치 버튼, 알림 진입 버튼 포함
- 모바일에서는 바텀시트, 데스크톱에서는 우측 패널 사용
- 상태색(안전/주의/위험)과 아이콘을 함께 사용

결과에는 레이아웃 구조, 컴포넌트 상태, 반응형 규칙을 포함하세요.
```

### 2-2. 위험 지역 상세 패널

```md
안전 탐사 지도 웹앱의 "위험 지역 상세 패널" UI를 설계하세요.
요구사항:
- 위험도 점수, 최근 신고 수, 최근 업데이트 시간, 권장 행동 요약 표시
- 긴급 행동 버튼(우회 경로 보기, 신고하기) 강조
- 정보 과부하를 막기 위해 섹션별 접기/펼치기 가능
- 데이터 신뢰도(공식/사용자 제보) 표시

결과에는 정보 우선순위, 시각 계층, 경고성 UI 톤 가이드를 포함하세요.
```

### 2-3. 안전 경로 탐색 화면

```md
안전 탐사 지도 웹앱의 "안전 경로 탐색 결과 화면"을 디자인하세요.
요구사항:
- 최단 경로보다 안전 경로를 우선 표시
- 경로별 위험 구간 개수/예상 시간/거리 비교
- 경로 변경 시 지도와 리스트가 동기화
- 경로 시작 CTA는 항상 화면 내 눈에 띄게 유지

결과에는 비교 카드 UI, 경로 하이라이트 방식, 실패 상태(경로 없음) 처리안을 포함하세요.
```

### 2-4. 제보 등록 화면

```md
안전 탐사 지도 웹앱의 "제보 등록 화면"을 설계하세요.
요구사항:
- 제보 위치 선택(지도 탭), 유형 선택, 상세 설명, 사진 첨부
- 입력 단계 최소화 (모바일 기준 1분 내 등록 목표)
- 민감 정보 입력 방지 안내 문구 포함
- 제출 후 처리 상태 추적 가능한 피드백 제공

결과에는 폼 UX, 유효성 검사 메시지, 성공/실패 피드백 패턴을 포함하세요.
```

## 4) Tailwind 구현 지시 프롬프트 (개발 연결용)

```md
아래 디자인 시안을 Tailwind CSS 기반 React 컴포넌트로 변환하세요.
조건:
- 클래스는 의미 단위로 정리하고 중복 최소화
- 상태(hover/focus/active/disabled/error)를 모두 정의
- 접근성 속성(aria-label, role, 키보드 포커스) 반영
- 모바일 우선 반응형으로 작성
- 지도 오버레이 패널은 z-index 충돌 없이 설계
- 애니메이션은 transition 150~250ms 범위 내 사용

출력:
1) 컴포넌트 구조
2) Tailwind 클래스 예시
3) 재사용 가능한 UI 패턴
4) 추후 디자인 시스템화 포인트
```

## 5) 프롬프트 품질 체크리스트

- 도메인(안전/재난/지도) 맥락이 포함되어 있는가?
- 사용자 역할(시민/운영자/관리자)이 반영되어 있는가?
- 상태색과 접근성 규칙이 명확한가?
- 모바일/데스크톱 반응형 규칙이 있는가?
- 실제 개발(Tailwind)로 연결 가능한 산출물 형식인가?

## 6) 금지사항 (디자인 가드레일)

- 위험 상태를 화려한 효과로 장난스럽게 표현하지 않기
- 텍스트 대비가 낮은 연한 회색 남용 금지
- 색상만으로 상태 전달 금지 (아이콘/라벨 병행)
- 지도 핵심 정보 가리는 과도한 플로팅 UI 금지
- 긴급 액션 버튼을 화면 하위 우선순위에 배치 금지
